# User Stories

## Персоны
- **Неавторизованный пользователь** — пользователь, не вошедший в аккаунт веб-приложения
- **Авторизованный пользователь** — пользователь, вошедший в аккаунт веб-приложения
- **Requester** — пользователь, который создает запросы и отправляет их на рассмотрение
- **Approver** — пользователь, который рассматривает запросы и принимает решение

---

## US-001: Регистрация пользователя
**Как** неавторизованный пользователь  
**Я хочу** зарегистрироваться в веб-приложении через администратора компании  
**Чтобы** получить доступ к функциям системы

### Критерии приёмки
- [ ] Пользователь может отправить запрос на регистрацию
- [ ] Администратор создает учетную запись пользователя
- [ ] Пользователь получает данные для входа
- [ ] После регистрации пользователь может авторизоваться в системе

---

## US-002: Авторизация
**Как** неавторизованный пользователь  
**Я хочу** авторизоваться в веб-приложении  
**Чтобы** пользоваться его функциями

### Критерии приёмки
- [ ] На странице входа есть поля для логина и пароля
- [ ] Пользователь может войти только с корректными данными
- [ ] При неверных данных отображается сообщение об ошибке
- [ ] После успешного входа пользователь попадает на главную страницу системы

---

## US-003: Выход из аккаунта
**Как** авторизованный пользователь  
**Я хочу** выйти из аккаунта  
**Чтобы** завершить работу в системе и закрыть доступ к функционалу

### Критерии приёмки
- [ ] В интерфейсе доступна кнопка выхода
- [ ] После выхода текущая сессия завершается
- [ ] После выхода пользователь перенаправляется на страницу входа
- [ ] Без повторной авторизации доступ к защищенным страницам невозможен

---

## US-004: Восстановление пароля
**Как** неавторизованный пользователь  
**Я хочу** восстановить пароль через администратора компании  
**Чтобы** снова получить доступ к своему аккаунту

### Критерии приёмки
- [ ] Пользователь может запросить восстановление пароля
- [ ] Администратор может сбросить пароль пользователя
- [ ] Пользователь получает новый пароль или инструкцию по смене пароля
- [ ] После восстановления пользователь может войти в систему

---

## US-005: Просмотр списка запросов
**Как** Requester / Approver  
**Я хочу** видеть список моих или полученных запросов  
**Чтобы** просматривать и управлять ими

### Критерии приёмки
- [ ] Requester видит только созданные им запросы
- [ ] Approver видит только запросы, направленные ему на рассмотрение
- [ ] В списке отображаются основные данные запроса
- [ ] Пользователь может открыть любой доступный ему запрос из списка

---

## US-006: Поиск запросов
**Как** Requester / Approver  
**Я хочу** искать запросы по заданным критериям  
**Чтобы** быстро находить нужные заявки

### Критерии приёмки
- [ ] Доступен поиск по статусу заявки
- [ ] Доступен поиск по типу запроса
- [ ] Доступен поиск по имени создателя
- [ ] Результаты поиска обновляются в списке запросов

---

## US-007: Сортировка запросов
**Как** Requester / Approver  
**Я хочу** сортировать запросы по определенным критериям  
**Чтобы** удобнее анализировать список заявок

### Критерии приёмки
- [ ] Доступна сортировка по общей цене
- [ ] Доступна сортировка по времени последнего изменения
- [ ] Доступна сортировка по имени создателя
- [ ] Пользователь может менять порядок сортировки

---

## US-008: Просмотр деталей запроса
**Как** Requester / Approver  
**Я хочу** открыть запрос  
**Чтобы** детально изучить его содержимое

### Критерии приёмки
- [ ] Пользователь может открыть карточку запроса
- [ ] В карточке отображаются все основные поля запроса
- [ ] Отображается текущий статус запроса
- [ ] Отображается информация о создателе и сумме запроса

---

## US-009: Создание запроса
**Как** Requester  
**Я хочу** создать новый запрос  
**Чтобы** отправить его на рассмотрение

### Критерии приёмки
- [ ] Requester может открыть форму создания запроса
- [ ] В форме доступны обязательные поля для заполнения
- [ ] Система проверяет заполнение обязательных полей
- [ ] После сохранения запрос получает статус "NEW"
- [ ] После создания запрос появляется в списке запросов пользователя

---

## US-010: Редактирование запроса
**Как** Requester  
**Я хочу** редактировать детали запроса, пока он не рассмотрен  
**Чтобы** исправлять или дополнять информацию

### Критерии приёмки
- [ ] Requester может редактировать только свои запросы
- [ ] Редактирование доступно только для нерассмотренных запросов
- [ ] После сохранения изменения отображаются в карточке запроса
- [ ] История и статус запроса сохраняются корректно

---

## US-011: Удаление запроса
**Как** Requester  
**Я хочу** удалить запрос  
**Чтобы** он больше не участвовал в процессе рассмотрения

### Критерии приёмки
- [ ] Requester может удалить только свой запрос
- [ ] Удаление доступно только для нерассмотренных запросов
- [ ] Перед удалением система запрашивает подтверждение
- [ ] После удаления запрос исчезает из списка

---

## US-012: Одобрение запроса
**Как** Approver  
**Я хочу** одобрить запрос  
**Чтобы** отметить его как "Approved" и передать на исполнение

### Критерии приёмки
- [ ] Approver может открыть запрос, направленный ему на рассмотрение
- [ ] Для Approver доступна кнопка одобрения
- [ ] После одобрения статус запроса меняется на "APPROVED"
- [ ] Requester видит обновленный статус запроса

---

## US-013: Отклонение запроса
**Как** Approver  
**Я хочу** отклонить запрос с указанием причины  
**Чтобы** отметить его как "Rejected" и вернуть Requester

### Критерии приёмки
- [ ] Для Approver доступна кнопка отклонения
- [ ] При отклонении необходимо указать причину
- [ ] После отклонения статус запроса меняется на "REJECTED"
- [ ] Requester видит причину отклонения в деталях запроса

---

## US-014: Повторная отправка запроса
**Как** Requester  
**Я хочу** отредактировать отклоненный запрос и отправить его повторно  
**Чтобы** снова передать его на рассмотрение

### Критерии приёмки
- [ ] Requester может открыть отклоненный запрос
- [ ] Requester может изменить его данные
- [ ] После повторной отправки запрос получает статус "RESUBMITTED" 
- [ ] Запрос снова появляется у Approver для рассмотрения

---

## US-015: Конвертация валют
**Как** Requester / Approver  
**Я хочу** выбрать удобную для меня валюту  
**Чтобы** суммы запросов автоматически отображались в нужной валюте

### Критерии приёмки
- [ ] Пользователь может выбрать предпочитаемую валюту
- [ ] Система автоматически конвертирует суммы запросов
- [ ] Исходная сумма запроса сохраняется корректно
- [ ] Отображаемая сумма обновляется в интерфейсе после смены валюты
